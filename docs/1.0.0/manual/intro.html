EProperties (Extended-Properties) extends the function of the standard java.util.Properties object.

Why?

Why. It seems a fair question...

Software should be configurable. As Java developers, writing increasingly complex software systems, the need for complex configurations has grown and continues to grow. Still methods of configuring complex systems are limited and lacking. There are two primary methods of configuring complex systems today:

Java Properties Files

Java properties, in the java.util package, provide a simple syntax for defining key/value pairs - in many ways the essence of configuration. Properties in this form were included in Java 1.0, and except for adding an XML based representation, they have not changed since that release.

Drawbacks:

  • limited to a 'flat' structure
  • no means for complex structures
  • no support for substitutions

Anyone who's seen a complex log4j.properties understands how difficult it can be to add structure to simple java.util.Properties files.

Custom XML Configuration Files

Many would say that the answer to this lack of capability is using XML files for configuration. However this has its drawbacks as well:

  • Need a new syntax for every configuration option
  • verbose and often not suitable for simple configurations
  • Not suitable for the many objects/subsystems that already use java.util.Properties for configuration.

Both of these configuration options seem lacking. EProperties offers an alternative with a simple structural extension to the well known java.util.Properties syntax, while not incurring the inflexibility and verbosity of XML based configurations.

EProperties defines a simple, intuitive, lightweight syntax that should 'feel' familiar to an experienced Java developer. This syntax supports many features including variable substitution, nesting, lists, and inclusion. This syntax and rich feature set frees developers from the limitations of java.util.Properties, without imposing the overhead and limitations of XML based configuration schemes.

EProperties is a simple and powerful means of introducing comprehensive configuration to very complex systems.

Survey of Other Extensions / Solutions

EProperties is not the first attempt to 'Extend' the existing properties API. There are other alternatives, and the developer should consider these options when evaluating EProperties. We feel after comparison, EProperties can stand on its rich feature set, focus, and concise API.

ExtendedProperties in Commons Collections

ExtendedProperties

This is a simple - and likely quite useful - extension to the Properties object. Active development seems to have ceased around 2004, and it has been replaced by Commons Configuraion.

Commons Configuration

http://commons.apache.org/configuration/

Commons Configuration is a truly comprehensive configuration utility, and as its name implies, it is more than simply an extension of the java.util.Properites. This platform supports merging of XML and Properties based configuration, as well as inclusion and substitution. And as an Apache project is merits consideration. However it seems overly complex, and lacking focus at times.
