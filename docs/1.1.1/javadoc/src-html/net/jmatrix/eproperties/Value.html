003    /** 
004     * This interface is used internally, to return the representation of the 
005     * object that should be saved, not the runtime value.  The difference
006     * between runtime value and persistence value is exactly substitutions.
007     *
008     * This concept is best shown by example. Consider the following 
009     * properties file:
010     * <pre>
011     * foo="world"
012     * bar="hello ${foo}"
013     * </pre>
014     * 
015     * Here we have 2 keys, and 2 values. The runtime and persistent values
016     * represented by the key "foo" are identical.  This is not the case for
017     * bar.  For bar, the following values are represented:
018     * 
019     * bar runtime value    = hello world
020     * bar persistent value = hello ${foo}
021     * 
022     * We should never persist the runtime value.  <p>
023     * 
024     * Further, internally, the key bar must listen to the key foo for changes.
025     * Substitutions can be done dynamically, however we also need to listen
026     * to foo because others may be listening to us.  Ie, if I declare myself
027     * to be a listener of the key "bar", and someone changes the value of
028     * "foo" then I need to be notified that my runtime value has also changed.
029     * 
030     * Substitutions could be done dynamically (as in when getRuntimeValue() is 
031     * called), or could be done statically - and listen for updates up the 
032     * tree. Either mechanism may be used, however changes to "foo" must be
033     * immediately reflected in "bar".  
034     */
035    public interface Value<T> {
036       // Value - concept and implementations coded 26 Nov 2008, B757 ATL-SEA
037       public T getPersistentValue();
038       public Object getRuntimeValue();
039    }
